DCS Auto-GCI

Standalone GCI / Threat Warning companion for DCS World

DCS Auto-GCI is a Windows application that runs alongside DCS World and provides real-time threat awareness. It detects enemy aircraft, SAM sites, AAA, and incoming missiles — then displays them on screen and announces them via text-to-speech with clock position, distance, and altitude.

Features

Enemy aircraft detectionBandits within ~80 nm shown with clock position, distance, altitude
SAM / AAA detectionGround threats displayed with bearing and range
Missile warningIncoming missiles announced immediately
Mission eventsShot launches, hits, and gun fire detected (with optional hook script)
Text-to-speechSpoken callouts like "Warning! Missile, 3 o'clock, 12 miles"
Voice & device selectionChoose any installed Windows SAPI5 voice and audio output device
Visual threat tableColor-coded: Red = Weapons Orange = Air Yellow = Ground

Integrated settingsAll options in a built-in settings panel — no separate dialogs

Requirements

  • DCS World — any edition (Stable or Open Beta)
  • Windows 10 or 11 — SAPI5 text-to-speech is built into Windows

If running from source instead of the installer:

  • Python 3.8+
  • comtypes — pip install comtypes

Installation

Option A: Setup Installer (Recommended)

  1. Run DCS_AutoGCI_Setup.exe
  2. Click Install — the installer automatically:
    • Places DCS Auto-GCI.exe in your local app data
    • Installs the DCS Lua scripts to Saved Games\DCS\Scripts and Saved Games\DCS.openbeta\Scripts
    • Creates Desktop and Start Menu shortcuts
  3. Launch DCS Auto-GCI from the Desktop or Start Menu

Already have an Export.lua? (e.g., for SRS, Helios, DCS-BIOS)
Open your existing Export.lua in a text editor and paste the code below at the end of the file. The script is designed to chain with existing export callbacks.

Code to copy & paste into your existing Export.lua:

-- ============================================================================
--  DCS Auto-GCI — Export Script (append this to your existing Export.lua)
-- ============================================================================
do
    local TW_HOST     = "127.0.0.1"
    local TW_PORT     = 9876
    local TW_INTERVAL = 0.1
    local TW_MAX_RANGE_M = 150000
    local TW_MAX_WPN_M   = 300000

    local tw_udp, tw_started = nil, false

    local function tw_initSocket()
        package.path  = package.path  .. ";.\\LuaSocket\\?.lua"
        package.cpath = package.cpath .. ";.\\LuaSocket\\?.dll"
        local ok, s = pcall(require, "socket")
        if not ok then return nil end
        return s
    end

    local function tw_send(msg)
        if tw_udp then pcall(function() tw_udp:send(msg) end) end
    end

    local function tw_isEnemy(a, b)
        if a == nil or b == nil or a == b then return false end
        if b == 0 or b == "0" or b == "" or b == "Neutral" or b == "Neutrals" then return false end
        return true
    end

    local function tw_approxDistM(lat1, lon1, lat2, lon2)
        local dlat = lat1 - lat2
        local dlon = (lon1 - lon2) * math.cos(math.rad((lat1 + lat2) / 2))
        return math.sqrt(dlat*dlat + dlon*dlon) * 111320
    end

    local tw_prevStart     = LuaExportStart
    local tw_prevStop      = LuaExportStop
    local tw_prevNextEvent = LuaExportActivityNextEvent
    local tw_prevBefore    = LuaExportBeforeNextFrame
    local tw_prevAfter     = LuaExportAfterNextFrame

    LuaExportStart = function()
        local s = tw_initSocket()
        if s then
            tw_udp = s.udp(); tw_udp:settimeout(0)
            tw_udp:setpeername(TW_HOST, TW_PORT)
            tw_send("STATUS:CONNECTED"); tw_started = true
        end
        if tw_prevStart then tw_prevStart() end
    end

    LuaExportStop = function()
        if tw_started then
            tw_send("STATUS:DISCONNECTED")
            if tw_udp then pcall(function() tw_udp:close() end) end
            tw_udp, tw_started = nil, false
        end
        if tw_prevStop then tw_prevStop() end
    end

    LuaExportActivityNextEvent = function(t)
        if not tw_started then
            if tw_prevNextEvent then return tw_prevNextEvent(t) end
            return t + 1.0
        end
        local selfOk, selfData = pcall(LoGetSelfData)
        if selfOk and selfData then
            tw_send(string.format("SELF:%s|%.6f|%.6f|%.1f|%.4f",
                selfData.Name or "Unknown",
                selfData.LatLongAlt.Lat, selfData.LatLongAlt.Long,
                selfData.LatLongAlt.Alt, selfData.Heading))
            local myCoal, myLat, myLon = selfData.Coalition,
                selfData.LatLongAlt.Lat, selfData.LatLongAlt.Long
            local wOk, objects = pcall(LoGetWorldObjects)
            if wOk and objects then
                for id, obj in pairs(objects) do
                    if tw_isEnemy(myCoal, obj.Coalition) and obj.Type and obj.LatLongAlt then
                        local lvl1, cat, maxR = obj.Type.level1, nil, TW_MAX_RANGE_M
                        if lvl1 == 4 and obj.Type.level2 ~= 1 then
                            cat, maxR = "WEAPON", TW_MAX_WPN_M
                        elseif lvl1 == 1 then cat = "AIR"
                        elseif lvl1 == 2 then cat = "GROUND" end
                        if cat then
                            local d = tw_approxDistM(myLat, myLon,
                                obj.LatLongAlt.Lat, obj.LatLongAlt.Long)
                            if d < maxR then
                                tw_send(string.format(
                                    "THREAT:%s|%s|%s|%.6f|%.6f|%.1f|%.4f|%d",
                                    cat, tostring(id), obj.Name or "Unknown",
                                    obj.LatLongAlt.Lat, obj.LatLongAlt.Long,
                                    obj.LatLongAlt.Alt, obj.Heading or 0,
                                    type(obj.Coalition)=="number" and obj.Coalition or 0))
                            end
                        end
                    end
                end
            end
            local twsOk, twsInfo = pcall(LoGetTWSInfo)
            if twsOk and twsInfo and twsInfo.Targets then
                for i, tgt in ipairs(twsInfo.Targets) do
                    if tgt.Position then
                        tw_send(string.format("THREAT:AIR|tws_%d|TWS Contact|%.6f|%.6f|%.1f|0|0",
                            i, tgt.Position.Lat or 0, tgt.Position.Long or 0, tgt.Position.Alt or 0))
                    end
                end
            end
            local ltOk, locked = pcall(LoGetLockedTargetInformation)
            if ltOk and locked and locked.LatLongAlt then
                tw_send(string.format("THREAT:AIR|locked_0|Locked Target|%.6f|%.6f|%.1f|0|0",
                    locked.LatLongAlt.Lat or 0, locked.LatLongAlt.Long or 0, locked.LatLongAlt.Alt or 0))
            end
        end
        local nextTime = t + TW_INTERVAL
        if tw_prevNextEvent then
            local ot = tw_prevNextEvent(t)
            if ot and ot < nextTime then nextTime = ot end
        end
        return nextTime
    end

    LuaExportBeforeNextFrame = function()
        if tw_prevBefore then tw_prevBefore() end
    end
    LuaExportAfterNextFrame = function()
        if tw_prevAfter then tw_prevAfter() end
    end
end

Uninstalling

Two ways to uninstall:

  1. Run DCS_AutoGCI_Setup.exe again — it will show an Uninstall button
  2. Go to Windows Settings → Apps → DCS Auto-GCI → Uninstall

The uninstaller removes:

  • The application executable and shortcuts
  • All Lua scripts from DCS Saved Games folders
  • The settings file (autogci_settings.json)

How It Works

┌─────────────────────┐ UDP (localhost:9876) ┌─────────────────────┐ │ DCS World │ ───────────────────────────► │ DCS Auto-GCI │ │ │ Player pos, threats, │ (Python app) │ │ Export.lua polls │ mission events │ │ │ LoGetSelfData() │ │ • Bearing/distance │ │ LoGetWorldObjects()│ │ • Clock position │ │ │ │ • GUI display │ │ Hook script sends │ │ • TTS callouts │ │ shot/hit events │ │ │ └─────────────────────┘ └─────────────────────┘

The DCS Export script runs inside DCS World and polls player and world data 10 times per second. It sends JSON-formatted threat data via UDP to the Auto-GCI app on localhost:9876. The Python app receives the data, calculates bearings and clock positions relative to the player, and displays threats in a color-coded table while announcing them via text-to-speech.

Hook Script Setup (Optional)

The hook script provides mission event detection (missile launches, hits, etc.) on top of the polling-based detection. It needs LuaSocket available in the mission environment:

  1. Navigate to your DCS installation folder
  2. Edit Scripts\MissionScripting.lua
  3. Find and comment out these two lines (add -- in front): 
    --sanitizeModule('os')
--sanitizeModule('io')
This is the same change required by MOOSE, MIST, and other DCS scripting frameworks. If you already use those, you likely have this done.

Adding More TTS Voices

The app lists every SAPI5 voice installed on your Windows system. By default you get Microsoft David (male) and Microsoft Zira (female).

Method 1: Windows Speech Settings

  1. Open Settings → Time & Language → Speech
  2. Under "Manage voices", click Add voices
  3. Select and install additional voices
  4. Restart the app — new voices appear in the dropdown

Method 2: Install Language Packs

  1. Open Settings → Time & Language → Language & Region
  2. Click Add a language and install a language pack (e.g., English UK adds Microsoft Hazel)
  3. Each pack adds one or more voices

Method 3: Unlock OneCore Voices (One-Click)

Windows 10/11 ships with ~20+ hidden "OneCore" voices used by Narrator and Cortana. These are higher-quality voices (Aria, Jenny, Guy, Ryan, etc.) that Microsoft doesn't expose to third-party SAPI5 apps by default.

DCS Auto-GCI can unlock them automatically:

  1. Open the app and go to the Settings panel
  2. Click the green ⚡ Unlock OneCore Voices button
  3. Click Yes on the Windows admin (UAC) prompt that appears
  4. The Voice dropdown refreshes automatically — select your new voice!
What happens behind the scenes: The app exports the OneCore registry keys, rewrites the paths from Speech_OneCore to Speech, and imports them into the standard SAPI5 location. This is the same process as the manual method below, fully automated.
Note: This requires a one-time administrator (UAC) prompt because it writes to HKEY_LOCAL_MACHINE. The app only adds keys — it never deletes or modifies existing voices.
Manual Method (if you prefer to do it by hand)

If the one-click button doesn't work for your system, you can do it manually:

Step 1 — Open Registry Editor as Administrator

Press Win + R to open the Run dialog, type regedit, and press Enter. Click Yes on the UAC prompt.

Step 2 — Navigate to the OneCore Voices

In the left pane of Registry Editor, expand the tree to:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Speech_OneCore\Voices\Tokens

You'll see subkeys for each hidden voice. Names hint at the language and voice:

Step 3 — Export the Voice Keys

You have two options:

  • Single voice: Right-click a specific voice key (e.g. MSTTS_V110_enUS_AriaM) → Export
  • All voices at once: Right-click the Tokens folder itself → Export (recommended — gets ~20 voices in one shot)

Save the .reg file somewhere easy to find (e.g. your Desktop). Name it something like OneCore_Voices.reg.

Step 4 — Edit the .reg File (Find & Replace)

This is the key step. You need to change the registry path from the OneCore location to the standard SAPI5 location.

  1. Right-click the .reg file you exported → Open with → Notepad
  2. Press Ctrl + H to open Find & Replace
  3. In Find what: type Speech_OneCore
  4. In Replace with: type Speech
  5. Click Replace All
  6. Save the file (Ctrl + S) and close Notepad
What this does: Every registry path in the file changes from ...\Speech_OneCore\Voices\Tokens\... to ...\Speech\Voices\Tokens\... — which is the standard SAPI5 location that apps like DCS Auto-GCI read from.

Step 5 — Import the Modified .reg File

Double-click the edited .reg file. Windows will ask for confirmation:

  1. Click Yes on the "Are you sure you want to continue?" prompt
  2. You should see: "The keys and values contained in [file] have been successfully added to the registry"

Step 6 — Verify & Restart DCS Auto-GCI

You can verify the import worked by checking in regedit:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Speech\Voices\Tokens

You should now see the OneCore voice keys (e.g. MSTTS_V110_enUS_AriaM) alongside the original voices (TTS_MS_EN-US_DAVID_11.0, TTS_MS_EN-US_ZIRA_11.0).

Now close and reopen DCS Auto-GCI. Open Settings — the unlocked voices (Aria, Guy, Jenny, Ryan, etc.) will appear in the Voice dropdown.

How to Undo

If you want to remove the unlocked voices later:

  1. Open regedit
  2. Navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Speech\Voices\Tokens
  3. Delete any keys that start with MSTTS_V110 (these are the ones you added)
  4. The original TTS_MS_EN-US_DAVID and TTS_MS_EN-US_ZIRA keys are untouched
Tip: The "bulk export" method (right-clicking the entire Tokens folder in Step 3) is the fastest approach. One export, one find-and-replace, one import — and you unlock all ~20 OneCore voices at once.

Building from Source

To rebuild the EXE and installer yourself:

  1. Install prerequisites:
    • Python 3.8+ with pip install pyinstaller comtypes
    • .NET SDK 8+ (dotnet --version)
    • WiX toolset: dotnet tool install --global wix
    • WiX extensions: wix extension add WixToolset.UI.wixext and wix extension add WixToolset.Bal.wixext
  2. Run build_exe.bat — builds the EXE and Setup.exe, packages docs and Lua scripts into a clean release\ folder
